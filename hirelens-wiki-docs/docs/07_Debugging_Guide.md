# HireLens 2.0 — Debugging Guide

> Populated incrementally. Every day file that introduces a new failure mode adds an entry here, not just in the day file — so recurring bugs are searchable in one place.

## Format for New Entries
```
### [Sprint-Day] Short error description
**Symptom:** what you see
**Cause:** root cause
**Fix:** exact steps
```
